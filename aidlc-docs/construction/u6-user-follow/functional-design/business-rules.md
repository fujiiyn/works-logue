# U6 User & Follow — Business Rules

## プロフィール

### BR-U01: 表示名バリデーション
- 1〜100文字（必須）
- 空白のみは不可
- 重複チェックなし（同名許可）

### BR-U02: ヘッドラインバリデーション
- 0〜60文字（任意）
- プロフェッショナル・ヘッドライン（一行の肩書き）

### BR-U03: 自己紹介バリデーション
- 0〜200文字（任意）

### BR-U04: 居住地バリデーション
- 0〜100文字（任意）
- 自由テキスト

### BR-U04b: SNSリンクバリデーション
- 全て任意
- 対象: X (Twitter), LinkedIn, Wantedly, 個人サイト
- 各フィールドはURL形式（https://で始まる）
- jsonb: `{ x?: string, linkedin?: string, wantedly?: string, website?: string }`

### BR-U05: アバター画像制約
- 最大ファイルサイズ: 2MB
- 許可形式: JPEG, PNG
- サーバー側でリサイズ: 256x256px（アスペクト比維持、中央クロップ）
- 古いアバターは上書き時に削除

### BR-U06: カバー画像制約
- 最大ファイルサイズ: 5MB
- 許可形式: JPEG, PNG
- サーバー側でリサイズ: 1200x340px（アスペクト比維持、中央クロップ）
- 古いカバーは上書き時に削除

### BR-U07: プロフィール編集権限
- 自分のプロフィールのみ編集可能
- BANされたユーザーは編集不可
- admin は管理画面から別途操作（U7）

### BR-U08: 削除/BANユーザーのプロフィール
- deleted_at != null → 404
- is_banned == true → 404（公開プロフィールからは非表示）

## フォロー

### BR-F01: 自己フォロー禁止
- UserFollow: follower_id == followee_id → 400 Bad Request
- DB レベルでも CHECK 制約あり

### BR-F02: 重複フォロー防止
- 既にフォロー済みの場合、何もせず成功を返す（冪等操作）
- PK制約 (user_id, planter_id) / (follower_id, followee_id) で DB レベルでも保証

### BR-F03: フォロー対象の存在チェック
- 対象ユーザー/Planter が存在しない → 404
- 対象が deleted/banned → 404

### BR-F04: 自動フォロー
- Seed 投稿時: 投稿者が Planter を自動フォロー（既存実装を維持）
- Log 投稿時: 投稿者が Planter を自動フォロー（既存実装を維持）
- 手動アンフォロー後に再度 Log を投稿した場合: 再度自動フォロー

### BR-F05: アンフォロー
- フォローしていない状態でアンフォローしても 204（冪等）
- 自動フォローされた Planter も手動でアンフォロー可能

### BR-F06: Planter フォロー UI（Seed/Louge 詳細）
- Seed/Sprout 詳細: stats 行で `[X logs] [Y contributors] [+ フォロー]` の順に等間隔（gap-3）で並べる
- Louge 詳細: 右サイドバーの貢献者パネル下部の stats 行で `[X logs · Y contributors] [+ フォロー]` を等間隔（gap-3）で並べる
- スタイル: アウトライン（teal ボーダー + teal テキスト、塗りなし）。フォロー中は薄い teal 背景でアクティブ感を示す
- ホバー時に色や文言を切り替えない（静的に「+ フォロー」/「フォロー中」のみ）
- アンフォローは確認ダイアログを出さず即座に解除（楽観更新）
- 未ログイン時はクリックで `/login` に誘導
- 自動フォロー済みの Planter はデフォルトで「フォロー中」表示
- D5: アンフォロー時は `is_manually_unfollowed=true` をセット（再Log投稿時の自動再フォローを抑止）
- 状態同期: SSR の planter API は認証ヘッダー無しで呼ばれるため `is_following` は常に false。クライアント側で初回マウント時に1回だけ認証付きで再取得し、楽観更新を上書きしない（再フェッチループ抑止）

## ナビゲーション

### BR-N01: ユーザー名 → プロフィール遷移
- すべての投稿者表示（Planter カード、Seed/Louge 詳細ヘッダー、Log 一覧、貢献者サイドバー、フォロー一覧モーダル）でユーザー名はクリック可能
- 遷移先: `/user/{id}`
- ホバー時に下線でクリック可能を示す
- AI 生成ログ（is_ai_generated=true）はリンク化しない

### BR-N02: プロフィール アクションボタンの排他表示
- 自分のプロフィール: 「編集」ボタンのみ表示
- 他人のプロフィール: 「フォロー」/「フォロー中」ボタンのみ表示（同じスロット）
- 両者を同時に表示しない

## フォロー中フィード

### BR-FF01: フォロー中フィードの構成
- フォロー中の Planter の更新（新規Log、状態変化）
- フォロー中のユーザーが投稿した新規 Seed
- 両方を updated_at 降順で混合表示
- 認証必須（未ログイン → ログイン画面へリダイレクト）

### BR-FF02: ページネーション
- cursor-based（既存フィードと同じパターン）
- 1ページ20件

## 投稿履歴

### BR-H01: Seed一覧
- 対象ユーザーが author の Planter を時系列降順で表示
- 全状態（seed/sprout/louge）を含む

### BR-H02: Log一覧
- 対象ユーザーが投稿した Log を時系列降順で表示
- 各 Log に紐づく Planter のタイトル・状態を付与

### BR-H03: 参加Louge
- Seed投稿者として開花した Planter + Log貢献者として開花した Planter
- 重複排除、bloomed_at 降順
- status = 'louge' のもののみ
