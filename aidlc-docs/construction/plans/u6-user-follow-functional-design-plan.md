# U6 User & Follow — Functional Design Plan

## Plan Overview

U6 はユーザープロフィール（表示・編集・投稿履歴・インサイトスコア）とフォロー機能（Planter/ユーザーフォロー、フォロー中フィード）を実装する。

---

## Steps

- [x] **Step 1**: ドメインエンティティ定義（User拡張、PlanterFollow、UserFollow）
- [x] **Step 2**: ビジネスロジックモデル定義
  - [x] 2a: プロフィール表示ロジック（公開/自分、統計情報集計）
  - [x] 2b: プロフィール編集ロジック（表示名・自己紹介・アバター・タグ更新）
  - [x] 2c: アバターアップロードロジック（Supabase Storage連携）
  - [x] 2d: フォロー/アンフォローロジック（User-User、User-Planter）
  - [x] 2e: フォロー中フィード取得ロジック
  - [x] 2f: 投稿履歴タブ取得ロジック（Seed一覧/Log一覧/参加Louge）
- [x] **Step 3**: ビジネスルール定義
  - [x] 3a: バリデーションルール（表示名、自己紹介、アバターファイル制約）
  - [x] 3b: フォロー制約（自己フォロー禁止、重複防止）
  - [x] 3c: 自動フォロー挙動の整理（Seed/Log投稿時の既存実装との連携）
  - [x] 3d: プロフィールアクセス制御（BAN/削除ユーザー、編集権限）
- [x] **Step 4**: フロントエンドコンポーネント設計
  - [x] 4a: UserProfilePage（プロフィールヘッダー、統計カード、タブ切り替え）
  - [x] 4b: ProfileEditPage（専用編集ページ — ヘッドライン・カバー画像・タグ・得意テーマ）
  - [x] 4c: FollowButton（フォロー/フォロー中状態切り替え）
  - [x] 4d: フォロー中フィードタブ（サイドバー「フォロー中」→ホームフィードのタブ連携）
  - [x] 4e: 右サイドバー（得意テーマ・バッジ・似た専門性のユーザー）
