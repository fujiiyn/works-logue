### Q1: ユニット粒度

上記6ユニットの粒度は適切ですか？ 特に以下の点：
- U3（Log & Score）と U4（Louge）は密結合だが、AI 記事生成の複雑さを考慮して分離しています
- U5（Feed & Search）は U2 の基本フィードを拡張する形です
- U6（User & Follow）はフォロー機能（優先度：低）を含みます

A) この6ユニットで良い
B) U3 と U4 を統合して5ユニットにしたい
C) U6 のフォロー機能を別ユニットに分離して7ユニットにしたい
D) その他（具体的に）

[Answer]:A

### Q2: DB スキーマの実装タイミング

U1（Foundation）で DB スキーマ（テーブル定義・マイグレーション）を **全量** 一括で作成する案です。理由：後続ユニットが DB 依存を気にせず実装に集中できる。

A) 全量一括（U1 で全テーブル作成）— 推奨
B) ユニットごとに必要なテーブルを段階的に追加

[Answer]:A

### Q3: Figma デザイン作成のタイミング

各ユニットの Code Generation 前に、そのユニットに必要な画面の Figma デザインを作成する方針で良いですか？

A) ユニットごとに必要な画面を事前に Figma で作成してから実装 — 推奨
B) 全画面を先に Figma で一括作成してから実装に入る
C) Figma は省略し、コードで直接 UI を作る

[Answer]:B

### Q4: 実装順序

依存関係に基づく推奨順序は U1→U2→U3→U4→U5→U6 です。ただし U5 と U6 は U3/U4 に依存しないため並行可能です。

A) 推奨順序（U1→U2→U3→U4→U5→U6）で良い
B) U5/U6 の順番を入れ替えたい（理由を記入）
C) その他の順番（具体的に）
