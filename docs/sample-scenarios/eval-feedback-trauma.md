# サンプルシナリオ: 評価面談で泣かれた件

開花エンジン / AIファシリテータの動作確認用、打ち込み台本。

「綺麗に整理された投稿」ではなく、現場で本当に困ってる感じ — 結論が出てない、感情がこもってる、断片的、共感ベース — を目指している。Seed主自身が会話の途中で視点を更新していくのも狙い。

---

## Seed

**投稿タイプ**: 悩み（Pain）

**タイトル**:
評価面談で部下に泣かれた。来週もう一回ある。何を伝えていいかわからない

**本文**:
来週、半期の評価面談がある。部下のひとりが、前回の評価で泣いてしまって、それ以来こっちもどう接していいかわからなくなった。

評価結果自体は前回とそんなに変わらない（特別良くも悪くもない）になりそうで、伝えるべきことも正直あんまり変わらない。でも前回みたいになるのが怖くて、フィードバックを丸めようかとも思ってる。それじゃダメなのもわかってる。

マネージャー2年目。評価面談の経験は5〜6回くらい。普段の1on1は普通にできてるつもりなんだけど、評価のときだけ部下との距離感がいきなり変わる気がする。

何をどう準備すればいいんだろう。

**タグ**:
- 役割: マネージャー/課長
- スキル: 目標設定、評価・フィードバック / コーチング
- 状況: メンタルヘルス
- 職種: HRBP相当ではないので「マネージャー業務」として「マネジメント」スキル

## Logs（投稿順）

> Seed主以外は別ユーザーから投稿する想定。Seed主自身も途中で再投稿する。

### Log 1（共感系・ふわっと）

わかります。私も最初の評価でやらかして、それ以来「相手の感情を傷つけない言い方」ばっかり考えるようになっちゃって、結局何も伝わらない面談になってる気がします。これ、本当に悩ましい。

### Log 2（具体的経験）

泣いたとき、その場でどう対応しました？うちは「一旦休憩しよう」って言って、お茶取りに行く間に部下も整理できたっぽくて、戻ってからの話はだいぶマシになった。一回止めるのも全然アリだと思います。

### Log 3（Seed主の追記）

自分も止めようとしたんですが、部下が「続けてください、大丈夫です」って言って、それで無理に続けたら最後まで泣きながら聞いてました。あれが一番後悔してる。

### Log 4（軽くカウンター）

それはキツい… でも「続けてください」って言われても、一回止める権限はマネージャー側にあると思いますよ。本人が大丈夫って言ってるから続行、は罠だと思う。

### Log 5（視点の転換・コア）

ちょっと違う角度かもですが、評価面談で泣くのって、その面談の話じゃなくて、それまでの1on1で言うべきだったことが溜まってるパターンが多い気がします。「なんで今さら言うんだ」感が爆発するというか。

### Log 6（Seed主・気づき）

…たしかに、前回も「3ヶ月前のあの案件で〇〇だった」って話を持ち出したのが引き金だった気がする。本人からしたら不意打ちだったのかも。

### Log 7（Log 5への補足）

それ重要だと思う。「評価面談で初めて聞かされる」のが地雷で、評価結果の数字そのものは別に二次的な話なのかも。

### Log 8（具体プラクティス）

ウチの上司は、面談の1週間前に「来週これとこれを話します」っていうメモを送ってきます。それで心の準備ができるし、こっちからも「この点は事前に補足したい」って返せる。サプライズなしにしてくれる人。

### Log 9（一歩引いた整理）

これって、評価面談の準備の話じゃなくて、普段の1on1の運用の話になってきてません？評価面談はあくまでアウトプットで、本当の課題はその前段にある気がする。

### Log 10（Seed主・方針更新）

おっしゃる通りで、頭が整理されてきました。来週の面談は「サプライズなし」を最優先にして、できれば前日までに話す論点を共有してみます。それと、泣き始めたら今度こそ一回止める。

### Log 11（最後の一押し）

それいいと思います。あと「評価のロジック」を先に話すのも効きます。「期初に〇〇を期待してた → それに対して △△ という結果だった → なのでこの評価」って構造で話すと、感情論にならずに済むことが多い。逆順で話すと「結論ありき」に聞こえる。

### Log 12（Seed主・締め）

ありがとうございます、めっちゃ助かりました。「評価そのもの」じゃなくて「事前の合意」と「論点の共有」が本丸って整理できただけで、来週への気の重さがちょっと減った気がする。やってみます。

## このサンプルの設計意図

| 観点 | 狙い |
|---|---|
| Seedの本文 | 「来週」「2年目」「5〜6回」など具体だが、結論が出ていない。Context は強いが Solution はない |
| Log 1〜4 | 共感・体験談ベース。まだ抽象化はされていない（情報の濃度はそれほど高くない） |
| Log 5 | **視点の転換**。「面談の問題ではなく、その前の1on1の問題」というリフレーム |
| Log 6 | Seed主が自分の体験と接続して気づき |
| Log 8〜11 | 具体プラクティス（事前メモ・ロジックを先に話す）→ Solution パーツが揃ってくる |
| Log 12 | Seed主自身が言語化したまとめ。Pattern Name っぽい一文（「事前合意 + 論点共有」） |

期待される構造パーツ充足:

- **Context**: Log 1〜3 で十分（マネージャー2年目、半期評価、過去に泣かれた件）
- **Problem**: Seed本文 + Log 5 で「事前共有がないまま重いフィードバックが飛ぶ」と特定
- **Solution**: Log 8 / 11 で「事前メモ」「ロジック先行」が出てくる
- **Name**: Log 12 で「事前合意 + 論点共有」として浮かぶ。AIが Louge 化するときに「サプライズなし評価面談」「事前合意型フィードバック」みたいなパターン名になるはず

開花条件A（structure_fulfillment == 1.0）が Log 11 あたりで初めて満たされ、そこから条件B（成熟度 0.9）に進む。Seed単独や Log 1 件では満たされない設計。

## 使い方

1. Seed投稿画面 (`/seed/new`) に上の Seed をコピペで投稿する
2. 別アカウント（`tanaka@test.works-logue.com` / `sato@test.works-logue.com` / `suzuki@test.works-logue.com`）で交互に Log を投稿する
3. ScoreCard の structure_parts チェックリストが一気に埋まらず段階的に増えていくか確認する
4. 途中で AI ファシリテータが介入するかも合わせて見る
