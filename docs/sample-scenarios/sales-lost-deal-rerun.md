# サンプルシナリオ: 失注した類似案件が、また最終提案フェーズに来てる件

開花エンジン / AIファシリテータの動作確認用、打ち込み台本（営業ver.）。

「綺麗に整理された投稿」ではなく、現場で本当に困ってる感じ — 結論が出てない、感情がこもってる、断片的、共感ベース — を目指している。Seed主自身が会話の途中で視点を更新していくのも狙い。

---

## Seed

**投稿タイプ**: 悩み（Pain）

**タイトル**:
半年追ってた商談が「価格」で失注した。来月、別案件で同じ顧客にまた最終提案。

**本文**:
来月、同じ業界・似たような規模感の大型商談がクロージング局面に入る。前回、半年追いかけた案件が最終提案の直前で他社に決まった。失注理由は「価格」って言われた。

提案内容は前回とそんなに大きくは変えていない（プロダクトも変わってないし、刺さってるユースケースも同じ）。でも、前回と同じパターンで負けるのが怖くて、最初から価格を10〜15%下げて出そうかと真剣に考えてる。それじゃダメなのは頭ではわかってる（粗利が死ぬし、値引き前提の商談になる）。

法人営業5年目。大型クロージングは5〜6回くらい経験ある。普段の中規模案件は普通に取れてる。でも、決裁が複数階層に上がる大型だけ、いきなり勝率が落ちる気がする。

そもそも前回の「価格で失注」って、本当に価格が理由だったんだろうか。なんか、それだけじゃない気もしてる。

何をどう準備すればいいんだろう。

**タグ**:
- 役割: 中堅
- 職種: 営業・カスタマーサクセス / フロント営業 / 法人営業（大手/中堅）
- スキル: 営業、交渉、仮説構築、提案
- 状況: （特になし、強いて言えば「炎上」一歩手前の自分の中の焦り）

## Logs（投稿順）

> Seed主以外は別ユーザーから投稿する想定。Seed主自身も途中で再投稿する。

### Log 1（共感系・ふわっと）

わかります。失注したあとに「価格でした」って言われると、次の案件で反射的に値引きを織り込みたくなるやつ。私も一回それやって、ディスカウントが標準になって戻れなくなりました。本当に悩ましい。

### Log 2（具体的経験）

失注理由のヒアリング、最後のメールやアンケートの回答だけで判断するの危険ですよ。うちは失注後2週間置いて、別ルート（窓口じゃない人）に「正直なところ何が決め手でした？」って雑談で聞いたら、全然違う答えが返ってきたことがあります。価格って言われてたけど、実際は「導入後の運用が想像できなかった」だった。

### Log 3（Seed主の追記）

それで言うと、前回の案件、最後に決裁者と直接話せてなかったかもしれない。窓口担当の方から「上で価格が引っかかってる」って聞いただけで、自分はその「上」と話せてないまま終わった。あれが今でも一番引っかかってる。

### Log 4（軽くカウンター）

それはキツい… でも、価格って言われた瞬間に値引き提案に走るのは罠だと思いますよ。「価格でした」って失注理由、受注側からすると一番伝えやすい言い訳ワードでもあるので、額面通り受け取るとずっと値引き合戦になる。

### Log 5（視点の転換・コア）

ちょっと違う角度かもですが、「価格で失注」って、価格そのものの話じゃなくて、その前段の「なぜ今、これに投資するか」の合意が顧客内で取れてないパターンが多い気がします。Why Now が弱いと、価格だけが浮いて見えるから、相対的に高く感じる。逆に Why Now が立ってる商談は、多少高くても通る。

### Log 6（Seed主・気づき）

…たしかに、前回の商談、「なぜ今やるか」を決裁者と一緒に作れてなかった。窓口の方とは「やった方がいいですよね」って温度では合ってたけど、その温度が上に伝わってたかと言われると怪しい。価格より前に、そっちが抜けてた気がしてきた。

### Log 7（Log 5への補足）

それ重要だと思う。「価格負け」は受注側の言い訳No.1ワードかもしれない。本当の原因は「課題の合意が浅い」「Why Now が立ってない」「決裁者リーチが弱い」のどれかで、それが価格という見えやすい指標に押し付けられてるだけ、ってケースが体感だとかなり多い。

### Log 8（具体プラクティス）

うちの先輩は、商談の中盤で必ず一回「仮にこの予算が通ったら、御社の何がどう変わりますか？」って聞いてます。これに具体で答えられない商談は、決裁通らないからクロージング無理せず引くって判断材料にしてる。逆にスラスラ答えが出てくる商談は、価格交渉になっても落ちにくい。

### Log 9（一歩引いた整理）

これって、失注分析の話というより、商談の組み立て順の話になってきてません？クロージングはあくまで最後のアウトプットで、本当の勝負は中盤までの「課題の握り方」「Why Now の握り方」「決裁者リーチ」にある気がする。

### Log 10（Seed主・方針更新）

おっしゃる通りで、頭が整理されてきました。来月の案件、いきなり価格を下げて出すのはやめます。代わりに、提案前にもう一度、決裁者にあたる方と「なぜ今これをやるべきか」を握り直す時間を取る。窓口だけで完結させない。それと、「仮に投資したら何が変わるか」を顧客側の言葉で言ってもらう質問を中盤に1回入れる。

### Log 11（最後の一押し）

それいいと思います。あと、提案の出し方の順番だけど、「課題の合意 → 解決アプローチの合意 → 投資判断（価格）」の構造で話すと、価格は最後の確認作業になる。逆順で「まず価格表お見せします」から入ると、商談ぜんぶが価格交渉に色付けされて、Why Now の議論がしにくくなる。

### Log 12（Seed主・締め）

ありがとうございます、めっちゃ助かりました。「価格負け」は本当に価格の話じゃなくて、「課題合意」と「Why Now」の握りが浅いことの言い換えだった、って整理できただけで、来月への気の重さがちょっと減った気がする。値引きから入るのはやめて、握り直しから始めます。

## このサンプルの設計意図

| 観点 | 狙い |
|---|---|
| Seedの本文 | 「来月」「5年目」「5〜6回」など具体だが、結論が出ていない。Context は強いが Solution はない。「価格で失注って本当か？」という違和感が伏線 |
| Log 1〜4 | 共感・体験談ベース。まだ抽象化はされていない。「価格って言われたら反射的に値引きしがち」までの一般論 |
| Log 5 | **視点の転換**。「価格の問題ではなく、その前段の Why Now の握りの問題」というリフレーム |
| Log 6 | Seed主が自分の体験と接続して気づき（決裁者と Why Now を握れてなかった） |
| Log 8〜11 | 具体プラクティス（中盤の質問、提案順序）→ Solution パーツが揃ってくる |
| Log 12 | Seed主自身が言語化したまとめ。Pattern Name っぽい一文（「課題合意 + Why Now 先行」） |

期待される構造パーツ充足:

- **Context**: Seed本文 + Log 3 で十分（法人営業5年目、大型クロージング経験あり、前回失注、決裁者リーチ未達）
- **Problem**: Seed本文 + Log 5 で「価格負けは表向きで、実態は Why Now と課題合意の不足」と特定
- **Solution**: Log 8 / 11 で「中盤での仮投資質問」「課題→解決→価格の順序」が出てくる
- **Name**: Log 12 で「課題合意 + Why Now 先行」として浮かぶ。AIが Louge 化するときに「Why Now 先行型クロージング」「価格負け＝合意不足の言い換え」みたいなパターン名になるはず

開花条件A（structure_fulfillment == 1.0）が Log 11 あたりで初めて満たされ、そこから条件B（成熟度 0.9）に進む。Seed単独や Log 1 件では満たされない設計。

1. Seed投稿画面 (`/seed/new`) に上の Seed をコピペで投稿する

4. 途中で AI ファシリテータが介入するかも合わせて見る
